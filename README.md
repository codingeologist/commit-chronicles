# Commit Chronicle
This repository contains a replayed history of my GitLab commits, preserving only the timestamps of my work. The commits are empty and use generic messages (`commit-chronicle-#1`, etc.) to avoid exposing original commit metadata.

### Why?
To reflect my development history and contribution rate on my GitHub profile, without including code, collaborators, or sensitive data.

### How?
1. Extracted commit timestamps from GitLab using `git log --author="MY_GITLAB_AUTHOR_NAME"`.
2. Replayed timestamps as empty commits to this repository, attributed to my GitHub account.
3. Pushed to GitHub to populate my contribution graph.

### Notes
- The commits are **not** representative of active development in this repository.
- The timestamps reflect my original work on GitLab, not contributions to this project.
